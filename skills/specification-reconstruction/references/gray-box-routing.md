# Gray-Box Routing

Use this reference only when the reconstruction workflow cannot describe an existing architectural seam confidently from repository evidence alone.

- Route that follow-on discovery to `derive-gray-box-boundaries`.
- Keep the handoff additive. Do not replace the derived stories, requirements, or technical design with gray-box analysis.
- If the extra boundary discovery changes the reconstructed design materially, update `derive-technical-design` output explicitly rather than leaving the new seam implicit.
